# 数据安全合规要求梳理平台 - 产品架构文档

> 版本：v1.0 | 日期：2026-04-17 | 状态：待确认

---

## 一、产品定位

**核心功能：** 给特定单位梳理其需要遵守的全部数据安全合规要求，输出可执行的责任清单。

**明确边界：**
- ✅ 本平台：合规要求梳理（应当做什么）
- ❌ 不在本平台：合规现状评估与匹配（做到了吗）

---

## 二、用户层

### 2.1 用户管理
- 飞书身份注册/登录
- 个人设置：通知偏好、时区

### 2.2 单位画像（Company Profile）

上传单位基本信息，系统自动判断适用法规：

| 字段 | 说明 | 影响判断 |
|------|------|---------|
| 单位名称 | 工商注册全称 | 搜索关联信息：处罚记录、新闻 |
| 所属行业 | 金融/医疗/教育/互联网/政务/交通/… | 直接决定 L5 行业标准适用范围 |
| 规模 | 大型/中型/小型/微型 | 影响部分法规的豁免条件 |
| 所在地 | 全国/省市 | 决定 L6/L7 地方文件适用范围 |
| 数据类型 | 个人信息/重要数据/核心数据/一般数据 | 决定保护义务等级 |
| 用户群体特征 | 是否有未成年人/老年人/患者等敏感群体 | 决定是否适用未成年人保护法、老年人权益保障法等 |
| 跨境业务 | 是/否 | 决定是否适用数据出境法规 |
| 上传材料 | 营业执照、业务说明、数据清单、隐私政策、安全评估报告等 | 作为合规匹配的依据材料 |

### 2.3 关联信息归集

根据单位名称，自动从全网搜索并归集：

| 信息类型 | 来源 | 用途 |
|---------|------|------|
| 监管处罚记录 | CAC/工信部/公安部/市监总局处罚公示、信用中国 | 判断该单位历史合规风险 |
| 行业监管动态 | 行业协会、监管机构公告 | 判断行业监管趋势 |
| 媒体相关报道 | 新闻搜索、微博、行业论坛 | 发现潜在合规舆情 |
| 同行处罚对比 | EDB执法案例库 | 对比同类违规的处罚力度 |
| 安全事件记录 | 国家漏洞平台、安全事件报告 | 判断历史安全事件 |

## 三、信息采集层

### 3.1 三种运行模式

| 模式 | 触发方式 | 场景 |
|------|---------|------|
| **定时模式** | 每日/每周固定时间自动跑 | 常规监控 |
| **手动模式** | 用户点击"立即更新" | 临时专项检查 |
| **后台自动发现** | 持续运行，发现新规立即入库+推送预警 | 重要新规不错过 |

### 3.2 信息发现渠道

每个层级（见第四章）均通过以下渠道发现信息：

| 渠道类型 | 示例 | 说明 |
|---------|------|------|
| **官方列表页** | CAC法规中心、SAMR国家标准系统 | 最权威、最完整 |
| **官方搜索API** | CAC JSON API（channel_code） | 可编程、全量 |
| **搜索引擎** | 百度/必应新闻搜索 | 发现舆情相关、行业动态 |
| **微信公众号** | 部委/省市官方公众号 | 征求意见稿首发渠道，经常比官网快 |
| **行业组织** | 金融数据标准化委员会、中国信通院 | 行业标准立项信息 |
| **研究机构** | 赛博研究院、电子标准院 | 白皮书、前沿研究 |
| **同行公开资料** | 上市公司合规报告、招股说明书 | 推断行业合规实践 |
| **内部文件线索** | 密级文件无网盘，但可通过文件名+简介先记录 | 标注"待获取/待核实" |

### 3.3 探索Agent（Research Agent）

独立运行的搜索 agent，持续探索：

- 维护已搜索过的关键词列表，避免重复
- 定期搜索立法动态、行业新闻、会议信息
- 发现新线索后记录：来源URL、摘要、相关性评分
- 探索微信公众号、微博热搜、行业论坛等非官方渠道
- 结果推送给 discovery agent 判断是否入库

## 五、真实性验证模块

### 5.1 验证流程

```
发现文件 → 来源可信度评估 → 文件完整性校验 → 版本一致性检查 → 入湖/标注
```

### 5.2 验证维度

| 维度 | 方法 | 结果 |
|------|------|------|
| **域名可信度** | 是否为 gov.cn/chinalaw.gov.cn/samr.gov.cn 等权威域名 | ✅可靠/⚠️待核实/❌可疑 |
| **文件完整性** | 下载后计算哈希（SHA256），与发布页面公示值对比 | 一致/不一致 |
| **版本确认** | 检查发文机关官网的版本说明，与入库版本对比 | 现行版/已修订/已废止 |
| **内容一致性** | 标题与内容是否对应 | ✅一致/⚠️存疑 |
| **内部/密级文件** | 无法在线获取全文的，标注"文件名+简介"，状态"待获取" | 线索记录 |

### 5.3 验证结果状态

- `已验证`：通过以上全部检查
- `待核实`：发现渠道可信度低或内容存疑，需要人工确认
- `待获取`：内部文件/密级文件，仅有线索
- `已失效`：发现新版本后自动标记

## 六、义务条款提取模块

### 6.1 提取内容

从法规正文中提取：

| 类型 | 示例 | 标记 |
|------|------|------|
| 义务性条款 | "应当"、"必须" | 橙色 |
| 禁止性条款 | "不得"、"禁止" | 红色 |
| 授权性条款 | "可以"、"有权" | 蓝色 |
| 处罚依据 | 对应处罚条款 | 紫色 |

### 6.2 结构化输出

```json
{
  "regulation_id": "L1-001",
  "article_number": "第四十二条",
  "obligation_type": "must",
  "content": "网络运营者应当采取技术措施和其他必要措施，确保其收集的个人信息不被泄露、篡改、毁损。",
  "obligation_keywords": ["采取技术措施", "保护个人信息"],
  "applicability": {
    "industries": ["互联网", "电信"],
    "scale": ["大型", "中型"],
    "data_types": ["个人信息"]
  }
}
```

## 七、适用性匹配引擎

### 7.1 输入：单位画像
- 所属行业 + 规模 + 所在地 + 数据类型 + 用户群体特征

### 7.2 输出：该单位的合规义务清单

```
适用法规 × 义务条款 → 合规动作清单（按优先级排序）
```

### 7.3 匹配规则

| 匹配维度 | 匹配逻辑 |
|---------|---------|
| 行业 | L3/L5 的"适用行业"字段与单位行业匹配 |
| 地域 | L6/L7 的"适用地区"与单位所在地匹配 |
| 规模 | L3/L5 的"适用规模"与单位规模匹配 |
| 数据类型 | 义务条款的关键词与单位数据类型匹配 |
| 用户群体 | 若含未成年人/老年人等，自动叠加对应保护法规 |
| 跨境业务 | 若有境外业务，自动叠加数据出境合规义务 |

## 八、引用关系图谱

### 8.1 关系类型

| 关系类型 | 方向 | 示例 |
|---------|------|------|
| 上位法→下位法 | L1 → L2 → L3 | 个人信息保护法 → 网络数据安全管理条例 → 部门规章 |
| 引用关系 | 法规 → 标准 | 个人信息保护法 → GB/T 35273 |
| 行业→国家 | L5 → L4 | JR/T 金融行业标准 → GB/T 数据安全标准 |
| 替代关系 | 旧版 → 新版 | 2020版个人信息安全规范 → 2025版 |
| 废止关系 | 原文件 → 废止文件 | XX办法 → 废止XX办法 |

### 8.2 可视化形式

- 独立的交互式图谱页面（可点击节点查看详情）
- 支持按层级、按行业、按发文机关筛选
- 支持查询某部法规的完整引用链和被引用链

## 九、任务与追踪管理

### 9.1 任务生成

基于合规义务清单，自动生成任务：

```json
{
  "task_name": "建立个人信息分类分级保护制度",
  "source_obligation": "L1-001 / 第四十二条",
  "due_date": "2026-06-30",
  "priority": "high",
  "status": "pending",
  "assignee": null,
  "related_regulations": ["L1-001", "L4-003"]
}
```

### 9.2 追踪状态

- 待处理 → 处理中 → 已完成 → 已验证
- 逾期提醒：到期前7天/3天/当天推送通知

## 十、预警与通知

| 预警类型 | 触发条件 | 推送方式 |
|---------|---------|---------|
| **新规预警** | 发现新法规入库 | 飞书消息 |
| **法规变更** | 已有法规被修订/废止 | 飞书消息 |
| **任务到期** | 距到期日7天/3天/当天 | 飞书消息 |
| **任务逾期** | 已超过到期日 | 飞书消息 |
| **监管动态** | 公司关联信息库发现新处罚/新闻 | 飞书消息 |

## 十二、产品里程碑计划

| 阶段 | 内容 | 说明 |
|------|------|------|
| **阶段一** | 产品架构定稿 | 本文档确认后锁定 |
